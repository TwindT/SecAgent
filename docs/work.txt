# SecAgent — 大模型驱动的安全智能分析系统

## 团队分工总览

项目按系统架构的四大模块进行分工，每人独立负责一个完整模块，端到端交付。模块间接口在第1周提前约定，之后各自并行开发。项目经理由模块2（后端服务）负责人兼任，统筹进度跟踪、文档整合与答辩准备。

---

### 智能分析可视化——Agent 交互体验与安全数据呈现

成员A负责整个前端项目的开发，技术栈为 React 18 + TypeScript + Ant Design + ECharts。核心工作包括：搭建仪表盘首页、任务提交页、分析过程页、报告展示页和历史管理页共六个页面；实现 Agent 思维链的实时动画展示组件（Thought → Action → Observe 三阶段渐入卡片），通过 WebSocket 接收后端推送的分析步骤并驱动前端状态机渲染；同时需要设计和实现安全数据的可视化图表，包括风险雷达图、漏洞分布图、ATT&CK 技术热力图等。这一模块的挑战在于将复杂的 AI 分析过程以直观、动态的方式呈现给用户。

---

### 分析服务中枢——后端架构与报告生成引擎

成员B负责后端服务开发并兼任项目经理，技术栈为 Python + FastAPI + SQLite + WebSocket。核心工作包括：设计并实现完整的 REST API（任务提交、状态查询、报告获取、历史管理等接口）；搭建 WebSocket 服务端，将 Agent 分析过程实时推送至前端；设计数据库表结构并管理分析结果的存储；实现 Markdown 报告模板引擎和 PDF 导出功能；以及文件上传的安全控制（类型白名单、大小限制、临时目录隔离）。作为项目经理，还需统筹整体进度、组织文档撰写和答辩 PPT 制作。

---

### 安全智能大脑——Agent 推理引擎与 LLM 决策调度

成员C负责 Agent 核心引擎的开发，技术栈为 Python + DashScope API + ReAct 范式。这是整个项目的"大脑"，核心工作包括：封装 DashScope API 调用（同步及流式输出）；实现 ReAct 主循环（Thought → Action → Observe），包含步数限制、超时控制、重试机制和兜底策略；设计 Function Calling 工具调度机制，管理工具注册、Schema 校验和调用结果反馈；编写 System Prompt 和 Few-shot 示例，定义 Agent 的安全分析师角色、行为边界和输出规范。这一模块的挑战不在代码量，而在于 Prompt 调优和循环逻辑的稳健性。

**【成员C个人任务概述】**

成员C负责 Agent 核心引擎的开发，这是整个项目的智能中枢。核心工作围绕大模型的调用与调度展开，首先需要封装大模型API的调用接口，支持常规请求和流式输出两种模式，确保系统能够灵活应对不同的交互场景。在此基础上实现自主推理循环，让系统能够像人类分析师一样交替进行思考和行动，根据当前分析状态自主决定下一步该调用哪个工具、如何解读返回结果、是否需要进一步深入分析。同时需要设计工具调度机制，将各类安全工具注册为可供调用的函数，并管理调用过程中的参数校验和结果反馈，使系统能够根据任务自动选择和组合工具。此外还要编写系统提示词和示例模板，定义系统的安全分析师角色定位、行为边界和输出规范，确保AI输出的专业性和一致性。在安全分析流程设计方面，需要制定代码漏洞检测和恶意文件分析的标准流程，明确每个环节的操作规范，同时制定安全分析的决策策略与置信度评估标准，设定系统的安全操作边界以防止执行危险操作。这一模块的挑战不在于代码量，而在于提示词效果的反复调优、推理循环逻辑的稳健性保障以及与各工具模块的协调对接。

---

### 安全能力底座——多源工具链集成与威胁情报融合

成员D负责安全工具链的集成与封装，技术栈为 Python + semgrep/bandit + 外部安全 API。核心工作包括：集成 semgrep 和 bandit 静态扫描工具，解析结果并结构化输出；实现文件分析功能，涵盖文件类型识别、PE 结构解析、脚本混淆检测、IOC 提取等；封装 NVD CVE 漏洞库和 AlienVault OTX/URLhaus 威胁情报的查询接口；加载 MITRE ATT&CK 知识库数据并实现攻击技术自动映射；同时为每个工具定义标准的 Function Schema，供 Agent 引擎调用。这一模块的特点是工具种类多、外部接口杂，需要做好统一的封装和格式化。
