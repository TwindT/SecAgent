## 1. 工具 Schema 标准格式

所有工具必须遵循统一的 Schema 格式，用于 LLM Function Calling。

### 1.1 工具定义结构

```json
{
  "type": "function",
  "function": {
    "name": "工具名称",
    "description": "工具功能描述（中文）",
    "parameters": {
      "type": "object",
      "properties": {
        "参数名": {
          "type": "参数类型",
          "description": "参数说明"
        }
      },
      "required": ["必填参数列表"]
    }
  }
}
```

### 1.2 工具清单

| 工具名称 | 功能描述 | 参数 | 返回值 |
|---------|---------|------|--------|
| `scan_code` | 代码静态扫描 | `code`: string, `language`: string | 漏洞列表 |
| `query_cve` | CVE 漏洞库查询 | `cve_id`: string / `keyword`: string | CVE 详情 |
| `query_threat_intel` | 威胁情报查询 | `ioc`: string, `type`: string | 情报结果 |
| `extract_file_features` | 文件特征提取 | `file_path`: string | 文件特征 |
| `extract_iocs` | IOC 提取 | `text`: string | IOC 列表 |
| `map_attack` | ATT&CK 技术映射 | `behavior`: string | 技术映射结果 |

### 1.3 工具返回格式

所有工具返回统一格式：

```json
{
  "success": true,
  "data": { /* 工具返回数据 */ },
  "message": "操作成功",
  "tool_name": "工具名称"
}
```

## 3. WebSocket 消息格式

### 3.1 连接地址

```
ws://localhost:8000/ws/{task_id}
```

### 3.2 消息类型

| 类型 | 说明 | 数据结构 |
|------|------|----------|
| `thought` | Agent 思考过程 | `{ "text": "思考内容", "step_num": 1 }` |
| `action` | 工具调用动作 | `{ "tool_name": "工具名", "params": {...}, "step_num": 1 }` |
| `observation` | 工具执行结果 | `{ "result": {...}, "step_num": 1 }` |
| `done` | 分析完成 | `{ "task_id": 1, "result_summary": {...} }` |
| `error` | 错误信息 | `{ "message": "错误描述", "code": "ERROR_CODE" }` |
| `progress` | 进度更新 | `{ "current_step": 3, "total_steps": 10 }` |

### 3.3 消息示例

**思考消息：**
```json
{
  "type": "thought",
  "data": {
    "text": "用户提交了一段 Python 代码，我需要分析其中是否存在安全漏洞。首先使用 semgrep 进行静态扫描。",
    "step_num": 1
  },
  "timestamp": "2026-06-09T10:30:00Z"
}
```

**动作消息：**
```json
{
  "type": "action",
  "data": {
    "tool_name": "scan_code",
    "params": {
      "code": "def login(username, password):\n    query = f\"SELECT * FROM users WHERE username='{username}' AND password='{password}'\"",
      "language": "python"
    },
    "step_num": 2
  },
  "timestamp": "2026-06-09T10:30:05Z"
}
```

**观察消息：**
```json
{
  "type": "observation",
  "data": {
    "result": {
      "success": true,
      "data": {
        "vulnerabilities": [
          {
            "rule_id": "sql-injection",
            "severity": "critical",
            "message": "SQL 注入漏洞"
          }
        ]
      }
    },
    "step_num": 2
  },
  "timestamp": "2026-06-09T10:30:10Z"
}
```

**完成消息：**
```json
{
  "type": "done",
  "data": {
    "task_id": 1,
    "result_summary": {
      "total_vulnerabilities": 1,
      "critical_count": 1,
      "high_count": 0
    }
  },
  "timestamp": "2026-06-09T10:35:00Z"
}
```

## 4. REST API 接口文档

### 4.1 基础路径

所有接口前缀：`http://localhost:8000/api`

### 4.2 接口列表

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/health` | 健康检查 |
| POST | `/tasks` | 创建分析任务 |
| GET | `/tasks` | 获取任务列表 |
| GET | `/tasks/{task_id}` | 获取任务详情 |
| POST | `/tasks/{task_id}/chat` | 发送追问消息 |
| GET | `/tasks/{task_id}/report/pdf` | 下载 PDF 报告 |
| POST | `/upload` | 文件上传 |

### 4.3 接口详细定义

#### 4.3.1 健康检查

**GET /health**

响应：
```json
{
  "status": "healthy",
  "timestamp": "2026-06-09T10:00:00Z"
}
```

#### 4.3.2 创建任务

**POST /api/tasks**

请求体：
```json
{
  "type": "vulnerability_detection",
  "input_content": "def login(username, password):\n    query = ...",
  "input_path": null
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | `vulnerability_detection` 或 `malware_analysis` |
| input_content | string | 否 | 代码内容（与 input_path 二选一） |
| input_path | string | 否 | 文件路径（已上传文件） |

响应：
```json
{
  "task_id": 1,
  "message": "Task created successfully",
  "status": "pending"
}
```

#### 4.3.3 获取任务列表

**GET /api/tasks**

查询参数：
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | int | 1 | 页码 |
| limit | int | 10 | 每页数量 |
| type | string | null | 任务类型筛选 |
| status | string | null | 状态筛选 |

响应：
```json
{
  "total": 100,
  "page": 1,
  "limit": 10,
  "tasks": [
    {
      "id": 1,
      "type": "vulnerability_detection",
      "status": "done",
      "created_at": "2026-06-09T10:00:00Z"
    }
  ]
}
```

#### 4.3.4 获取任务详情

**GET /api/tasks/{task_id}**

响应：
```json
{
  "id": 1,
  "type": "vulnerability_detection",
  "status": "done",
  "input_content": "...",
  "result_json": "{...}",
  "created_at": "2026-06-09T10:00:00Z",
  "updated_at": "2026-06-09T10:35:00Z",
  "analysis_steps": [...],
  "conversations": [...]
}
```

#### 4.3.5 发送追问消息

**POST /api/tasks/{task_id}/chat**

请求体：
```json
{
  "message": "这个漏洞的修复方案是什么？"
}
```

响应：
```json
{
  "task_id": 1,
  "reply": "针对 SQL 注入漏洞，建议使用参数化查询...",
  "timestamp": "2026-06-09T11:00:00Z"
}
```

#### 4.3.6 文件上传

**POST /api/upload**

Content-Type: `multipart/form-data`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 上传的文件 |

响应：
```json
{
  "filename": "sample.py",
  "file_path": "/tmp/uploads/sample.py",
  "size": 1024,
  "md5": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6"
}
```

## 5. 错误处理规范

### 5.1 错误响应格式

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述",
    "detail": "详细信息（可选）"
  },
  "timestamp": "2026-06-09T10:00:00Z"
}
```

### 5.2 错误码列表

| 错误码 | 说明 | HTTP 状态码 |
|--------|------|------------|
| `INVALID_INPUT` | 无效输入参数 | 400 |
| `TASK_NOT_FOUND` | 任务不存在 | 404 |
| `FILE_TOO_LARGE` | 文件超过大小限制 | 413 |
| `INVALID_FILE_TYPE` | 不支持的文件类型 | 400 |
| `ANALYSIS_ERROR` | 分析过程出错 | 500 |
| `RATE_LIMIT_EXCEEDED` | 请求频率超限 | 429 |

---

## 附录：状态码对照表

| HTTP 状态码 | 含义 |
|------------|------|
| 200 | 请求成功 |
| 201 | 创建成功 |
| 400 | 无效请求 |
| 404 | 资源不存在 |
| 413 | 请求体过大 |
| 429 | 请求频率超限 |
| 500 | 服务器内部错误 |
