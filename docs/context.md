## 1. 设计题目

**SecAgent — 基于大语言模型的AI安全智能分析Agent系统**

> 英文题目：SecAgent: AI-Powered Security Analysis Agent System Based on Large Language Models

---

## 2. 主要内容

本项目开发一个基于大语言模型的AI安全智能分析Agent系统，结合静态代码扫描、威胁情报查询与恶意文件分析等安全工具，实现对代码漏洞的自动化检测与可疑文件的智能研判，并通过思维链可视化技术展示分析全过程。

### 2.1 系统功能要求

本项目旨在开发一个融合人工智能与安全技术的智能分析Agent系统。系统主要包含以下功能模块：

**（1）智能代码漏洞检测**

系统支持用户通过Web界面上传或粘贴源代码，涵盖Python、Java、JavaScript、C等主流编程语言。接收到代码后，系统会自动调用semgrep、bandit等静态分析工具进行规则扫描，随后由AI引擎对扫描结果进行语义级深度分析，自动过滤误报并发现规则无法覆盖的逻辑漏洞。系统能够自动关联CVE漏洞库和CWE弱点分类，为每个漏洞标注类型、危险等级（高/中/低）及CWE编号，并生成结构化报告。报告内容包括漏洞所在位置（文件名与行号）、相关代码片段以及修复建议（附带代码示例）。系统主要检测的漏洞类型包括SQL注入、XSS跨站脚本、命令注入、路径遍历、硬编码密钥、反序列化漏洞、权限控制缺陷等常见安全问题。

**（2）恶意代码/文件智能分析**

系统支持上传可疑文件进行分析，包括PE可执行文件、ELF文件、脚本文件、Office文档等。上传后系统会自动识别文件类型并提取静态特征，如导入表、字符串、哈希值、可疑URL或IP地址等。AI引擎会对文件内代码逻辑进行语义分析，识别下载执行、信息窃取、持久化等恶意行为模式，同时自动提取IOC失陷指标（恶意IP、域名、URL、Hash）。系统还会对接AlienVault OTX、URLhaus等开源威胁情报平台进行信誉查询，并将分析结果映射到MITRE ATT&CK攻击技术框架，最终输出恶意判定结果（恶意/可疑/安全）及相应的置信度百分比。

**（3）Agent自主决策与思维链可视化**

系统基于ReAct（推理-行动）范式构建Agent引擎，使Agent能够自主规划分析路径，包括决定使用哪些工具、何时深入分析以及何时查询外部知识库。整个分析过程通过WebSocket实时推送到前端，前端以思维链卡片动画的形式展示Agent的思考过程，依次呈现思考、工具调用、观察结果和下一步决策等步骤，让用户能够直观地了解AI的分析逻辑。

**（4）交互式安全分析对话**

系统支持用户在分析过程中或分析完成后进行追问，例如要求重点检查认证逻辑或给出替代修复方案等。Agent能够基于分析上下文和历史对话记忆进行多轮对话回复，保持对话的连贯性和针对性。

**（5）安全报告生成与导出**

系统能够自动生成结构化的Markdown格式安全分析报告，报告中包含风险雷达图、漏洞类型分布饼图、ATT&CK技术热力图等可视化图表，并支持PDF格式的导出和下载。

**（6）Web管理界面**

系统提供完整的Web管理界面，包括展示任务统计和最近分析概览的仪表盘首页、支持代码粘贴和文件上传及参数配置的任务提交页面，以及具备列表展示、搜索、筛选和详情查看功能的历史任务管理页面。

### 2.2 团队分工与协作

> 说明：四人每人同时涉及AI技术和安全技术两个维度，满足课程"AI+安全"的综合性要求。

1. 成员A（Agent引擎 + LLM推理核心）

   AI方面：负责封装DeepSeek大模型API调用、设计System Prompt、实现ReAct循环及Function Calling工具调度、Prompt模板管理与调优。

   安全方面：负责设计漏洞检测与恶意代码分析标准流程、制定决策策略与置信度评估标准、设定Agent安全操作边界。

   常规工作：编写Agent引擎核心代码、Prompt模板库、Function Calling Schema定义及测试用例。难度较高。
2. 成员B（安全工具链 + 工具集成与编排）

   AI方面：负责设计安全工具Function Schema、实现AI对工具结果的二次研判逻辑、设计工具返回结果格式化、实现漏洞类型自动分类与ATT&CK技术映射。

   安全方面：负责集成semgrep/bandit静态扫描工具、封装NVD CVE漏洞库查询API、封装OTX/URLhaus威胁情报查询、实现文件特征提取、集成YARA规则、维护CWE与ATT&CK知识库。

   常规工作：编写工具链模块代码、工具Schema定义文件、CWE/ATT&CK数据文件及测试用例。难度中等。
3. 成员C（后端服务 + 报告生成引擎）

   AI方面：负责实现分析报告AI摘要生成、多轮对话管理、分析结果智能聚合排序、AI分析任务状态机管理与任务队列调度。

   安全方面：负责设计安全报告标准模板、实现数据脱敏处理、SQLite加密存储、文件上传安全控制、PDF报告安全导出、FastAPI后端安全配置。

   常规工作：编写FastAPI后端代码、数据库模型与初始化脚本、报告模板、Swagger API文档及WebSocket消息规范。难度中等。
4. 成员D（前端界面 + 数据可视化）

   AI方面：负责实现Agent思维链实时动画展示、设计AI分析过程用户交互体验、实现前端状态机、WebSocket实时接收并渲染分析进度。

   安全方面：负责设计实现安全数据可视化图表、漏洞信息展示组件、安全报告在线预览、安全配色体系与视觉设计。

   常规工作：编写React前端完整代码、可视化图表组件库、思维链动画组件、前端路由与页面及状态管理。难度中等。

**协作机制**

四人模块间的接口在第1周集中约定完成，包括工具Schema格式、数据库表结构、API文档、WebSocket消息格式以及分析结果JSON结构。成员A（Agent引擎）与成员B（工具链）通过标准化Tool Schema进行调用交互，成员A与成员C（后端服务）通过任务对象格式和分析结果数据结构进行对接，成员C与成员D（前端界面）通过REST API（Swagger自动生成）和WebSocket消息格式进行通信。团队使用Git进行版本管理，每人独立分支开发并定期合并。其中一人兼任项目经理（建议成员A或C），负责进度跟踪、文档整合、PPT制作及演示排练。

### 2.3 开发语言

**主开发语言：Python 3.10+**

选择Python作为主开发语言主要基于以下考虑：Python拥有丰富的AI/LLM生态（如openai SDK、langchain等），便于调用大模型API；同时Python也是安全领域的主流语言，semgrep、bandit、pefile、YARA等安全工具均提供Python接口；此外Python开发效率较高，适合6周的原型项目，且团队本科生对Python最为熟悉。

**前端语言：TypeScript + React 18**

前端采用TypeScript与React 18的组合，TypeScript能够提供类型安全，减少开发错误；React生态成熟，组件化开发可有效提高开发效率；Ant Design组件库提供专业的表格、表单、布局组件；ECharts能够满足安全数据可视化的需求。

> **注**：虽然模板提及C++/Java/C#，但本项目的性质和团队技术栈更适合Python+TypeScript方案。Python在AI与安全领域的生态优势远超其他语言，前端展示需要Web技术栈，这比传统桌面GUI开发语言更适合本项目的"Web全栈应用"定位。

### 2.4 开发环境

**开发工具**

IDE/编辑器选用Visual Studio Code作为主力工具（支持Python/Pylance及React/ESLint插件），辅以Cursor进行AI辅助编程。版本管理采用Git配合GitHub/Gitee私有仓库，API测试使用Postman或REST Client（VS Code插件），数据库管理使用DB Browser for SQLite，容器化开发依赖Docker Desktop与Docker Compose。

**运行环境**

操作系统支持Windows 11或Linux（Ubuntu 22.04），团队成员使用各自熟悉的系统，最终通过Docker统一环境。Python版本要求3.10以上，采用venv或conda进行虚拟环境管理。前端开发需要Node.js 18 LTS以上版本。数据库选用SQLite 3（内置，零配置轻量级），任务队列可选Redis 7以支持异步任务管理。

**AI服务（云端API）**

主模型采用DeepSeek-V3 API，该模型支持Function Calling且中文能力强，性价比较高。备选模型为阿里云通义千问Qwen-Max，国内访问稳定且有免费额度。

**安全工具依赖**

系统依赖的安全工具包括semgrep（多语言静态代码分析）、bandit（Python安全扫描）、pefile（PE文件分析）以及YARA-python（恶意代码特征匹配）。

**部署方案**

采用Docker Compose进行一键部署，包含前端Nginx容器、后端Python容器和Redis容器。前端静态文件由Nginx提供，后端FastAPI通过Uvicorn运行，通过docker-compose.yml统一编排，评审时可一键启动。

## 3. 进度安排

**项目总周期：6周（第1周～第6周）**

1. 第一周：完成需求分析与技术选型，搭建开发环境（Docker、前后端脚手架），设计数据库表结构，约定接口规范，整理CWE/ATT&CK知识库数据。
2. 第二至三周：开发Agent核心引擎（封装LLM API、实现ReAct循环、Function Calling工具调度），同步开发安全工具链（集成semgrep/bandit、封装CVE/威胁情报查询、文件分析工具），开发后端REST API与报告生成引擎。
3. 第四至五周：完成前端页面开发（仪表盘、任务提交、思维链动画展示、报告展示、历史任务管理），进行前后端联调与端到端测试，准备演示样本，修复Bug并优化性能。
4. 第六周：进行UI/UX细节打磨，撰写项目报告，制作PPT演示文稿，设计演示流程并排练，完成最终交付。

**关键里程碑**

1. 第一周末（M1）：基础框架就绪，前后端可通信、Docker一键启动。
2. 第三周末（M2）：Agent引擎可用，可自主完成一次完整代码漏洞分析。
3. 第四周末（M3）：后端功能完整，所有API与安全工具可被Agent调用。
4. 第五周末（M4）：系统联调通过，完整流程跑通。
5. 第六周末（M5）：项目正式交付，文档齐全、系统稳定。
